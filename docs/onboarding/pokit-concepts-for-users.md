# POKit 개념 한눈에 보기

POKit2 = 이슈 드리븐 하네스.

아이디어나 요청을 그냥 대화 속에 흘려보내지 않고, 작은 이슈로 잡고, 증거를 모은 뒤, 게이트에서 닫는 방식입니다.

```text
요청 -> 이슈 -> 실행 -> 검증 -> 게이트 -> 다음
```

복잡한 장치는 뒤에 접어둡니다. 처음에는 이렇게만 생각하면 됩니다.

```text
"무엇을 하고 싶은지 말한다"
  -> POKit이 작업 후보와 완료 기준을 정리한다
  -> 준비된 이슈 하나를 실행한다
  -> 검증 근거가 있어야 완료된다
```

**처음 시작한다면** 먼저 `npx pokit2 install` 한 줄로 설치하세요.
흐름과 명령어가 궁금하면 [pokit-flow-overview.md](./pokit-flow-overview.md)를 함께 보세요.

## 핵심 개념

| 개념 | 한줄요약 | 비유 | 흔한 오해 |
|---|---|---|---|
| 이슈 | 목표와 완료 기준을 담은 작업 단위 | 작은 주문서 | 대화 메모가 아니라, 완료 여부를 판단할 수 있는 약속입니다. |
| 다듬기 | 요청을 바로 만들기 전에 작업 후보와 질문으로 정리하는 단계 | 요리 전 재료 손질 | "생각만 하는 시간"이 아니라, 잘못된 실행을 줄이는 시작점입니다. |
| 검증 | 실제로 됐는지 확인할 증거를 모으는 단계 | 검사표 | 에이전트가 "됐어요"라고 말하는 것만으로는 검증이 아닙니다. |
| 게이트 | 모은 증거로 완료 판정을 내리는 단계 | 출고 전 최종 확인 | 테스트를 돌렸다고 자동 완료가 아닙니다. 게이트 통과가 따로 필요합니다. |
| 상태 외부화 | 현재 일, 다음 행동, 기억을 파일에 남기는 방식 | 작업대 위 보드 | 에이전트 머릿속 기억에 의존하지 않으려는 장치입니다. |
| 프로젝트 | 이슈 번호와 상태를 나누는 작업 공간 | 폴더별 노트 | 프로젝트를 안 만들면 기본 `common`에서 시작할 수 있습니다. |
| 세션과 worktree | 같은 이슈나 다른 이슈를 안전하게 나눠 작업하는 실행 공간 | 작업 책상 분리 | 여러 세션이 있다고 각자 마음대로 완료 처리하거나 push하는 것은 아닙니다. |
| 통합 | 나눠진 작업을 검토하고 받아들이는 단계 | 초안 승인 후 원본 반영 | 별도 작업 세션의 결과는 제안이고, 완료 판정은 통합 세션이 합니다. |
| 안내 카드 | 각 단계마다 러너가 현재 위치와 다음 행동을 카드(work / integration / status)로 보여주는 장치 | 진행 안내 표지판 | 카드는 표시 전용입니다. 카드가 뜬다고 단계가 자동 승인되지는 않습니다. |
| 안전 단계 자동화 🟢/🔴 | 되돌릴 수 있고 흔적이 남는 단계(코드 변경, 검증, 커밋)는 자동 진행 🟢, 외부 영향이나 완료 확정 단계(push, 게이트 통과)는 사람 확인이 필요 🔴 | 신호등 | push는 항상 PO 확인이 필요합니다. 🟢라도 근거 없이 건너뛰지 않습니다. |
| 스타터 안전 바닥 | 이슈 없이 파일을 바꾸려 하면 자동으로 막고, 이슈 초안을 제안합니다. 이슈 ID 앞 글자에 무관하게 동작합니다. | 출입증 없으면 입장 불가 | 안전 바닥은 `pokit install` 시 자동으로 설치되며, 기존 설정을 덮어쓰지 않고 병합합니다. |
| 멀티세션 조율 | 세션이나 worktree 여러 개가 열려 있을 때 `pokit issue:create` 등이 다른 세션이 이미 작업 중인 후보를 중복 선택하지 않도록 조율합니다. 세션은 시작 시 자동 등록됩니다. | 회의실 예약 시스템 | 로컬 훅은 advisory(—no-verify 우회 가능)이며, 최종 강제 적용은 서버 측에서 합니다. |
| Antigravity 런타임 | POKit의 네 가지 `/pokit.*` 기술을 Antigravity 런타임에서 `define_subagent` 계약 방식으로 에뮬레이션할 수 있습니다. 진입점은 `ANTIGRAVITY.md`입니다. | 외부 시스템 어댑터 | 공식 지원은 아직 deferred(스모크 통과, 전체 증거 미수집)입니다. 네이티브 지원으로 혼동하지 마세요. |

## 처음 쓰는 사람의 네 문장

```text
포킷 시작
```

현재 프로젝트, 진행 중인 이슈, 다음 행동을 복구합니다.

```text
이런 걸 만들고 싶어
```

POKit이 바로 코드를 쓰기보다 먼저 작업 후보, 완료 기준, 확인 질문을 정리합니다.

```text
진행하자
```

준비된 이슈를 실행합니다. 실행 전에는 수동/자동/중단을 고를 수 있습니다.

```text
다음
```

방금 이슈가 게이트를 통과했다면 다음 이슈로 넘어갑니다.

## 무엇이 편해지나

- 여러 요청이 섞여도 "지금 하는 일"이 하나로 보입니다.
- 완료 기준이 파일에 남아서 다음 세션이 이어받기 쉽습니다.
- 검증 없이 완료됐다고 주장하기 어렵습니다.
- 프로젝트를 나누면 이슈 번호와 상태도 프로젝트별로 관리됩니다.
- worktree 세션을 쓰면 동시 작업을 하되, 통합과 완료 판정은 한 곳에서 관리할 수 있습니다.
- 안내 카드가 각 단계에서 다음 행동을 알려줘서 흐름을 잃지 않습니다.
- 안전 단계 자동화 덕분에 되돌릴 수 있는 단계는 빠르게 진행되고, 위험한 단계에서만 멈춥니다.
- 이슈 없이 작업을 시작하려 하면 자동으로 막고 초안을 제안해서 실수를 줄입니다.

## v0.18 이전에 설치한 사용자에게

기존에 POKit을 전체 복사 방식(레포에 스크립트 사본 포함)으로 설치했다면, 한 번만 실행하면 됩니다.

```bash
pokit install
```

레거시 설치 형태를 자동으로 감지하고 새 방식으로 전환합니다. 이슈와 상태(`.ai-os`)는 그대로 보존되고, 본체 스크립트 사본만 제거됩니다.

## 접힌 기계

처음에는 몰라도 되는 내부 장치가 있습니다.

```text
러너 / 작업 나눔 / 기록 / 자동 점검 / 측정 기록 / 회고
안전 바닥 훅 / 세션 레지스트리 / Antigravity 에뮬레이션 계약
```

이 장치들은 사용자가 외울 단어가 아니라, POKit이 실수와 드리프트를 줄이기 위해 뒤에서 쓰는 구조입니다. 필요할 때만 문서나 로그에서 확인하면 됩니다.
